# 08-上线报告

> **负责人**：team-lead | **日期**：2026-06-16

## 上线内容

本次上线为 **Markdown 渲染修复**，解决了博客系统中文章内容无法正确渲染 Markdown 格式的问题。

### 修改清单

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `ArticleDetail.vue` | 重构 | 引入 `marked` 前端渲染，移除对 `contentHtml` 的依赖 |
| `ArticleService.java` | 重构 | 移除后端 `MarkdownUtil.renderToHtml()` 写入逻辑 |
| `ArticleDetailVO.java` | 修改 | 移除 `contentHtml` 字段 |
| `ArticleMapper.java` | 清理 | 移除 `updateContentHtml` 方法 |
| `AdminArticleController.java` | 清理 | 移除 `/re-render` 批量重渲染接口 |
| `types/index.ts` (frontend) | 修改 | 移除 `contentHtml` 类型定义 |
| `types/index.ts` (admin) | 修改 | 移除 `contentHtml` 类型定义 |

### 架构变更

```
改造前：content (MD) → 后端 renderToHtml() → content_html (HTML) → 数据库双字段
改造后：content (MD) → 数据库单字段 → 前端 marked.parse() → 页面渲染
```
