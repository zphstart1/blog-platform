# 09-交付报告

> **负责人**：team-lead | **日期**：2026-06-16

## 任务概述

**任务名称**：Markdown 渲染修复  
**任务类型**：Bug Fix + 架构优化  
**影响范围**：读者端文章详情页

### 问题描述

博客系统中文章使用 Markdown 编写，但读者端文章详情页无法正确渲染 Markdown 格式：
- `### 功能验证` 未转为 `<h3>` 标题
- `- 列表项` 未转为 `<ul><li>` 列表
- 内容以原始 Markdown 文本展示在 `<p>` 标签内

## 交付成果

### 1. 架构优化：双字段 → 单数据源

| 维度 | 改造前 | 改造后 |
|------|--------|--------|
| **存储** | `content`(MD) + `content_html`(HTML) 双字段 | 仅 `content`(MD) 单字段 |
| **渲染位置** | 后端 commonmark-java 写入时渲染 | 前端 marked 读取时渲染 |
| **一致性问题** | 旧数据未渲染、渲染逻辑升级后遗留 | 永不出现不一致 |
| **代码复杂度** | MD 检测 + 重渲染 + 批量修复 + 正则匹配 | 3 行 computed |

### 2. Bug 修复清单

| Bug | 根因 | 修复 |
|-----|------|------|
| `###` 不渲染为 `<h3>` | 数据库中 `\n` 为字面量转义序列 | 渲染前 `replace(/\\n/g, '\n')` normalize |
| `- ` 不渲染为 `<li>` | 同上 | 同上 |
| 文章背景为黑色 | `github-markdown.css` 含深色模式媒体查询 | 改用 `github-markdown-light.css` |
| 旧构建产物残留 | dist 未清理导致 Vite hash 不更新 | `rmdir /s /q dist` 后重建 |
