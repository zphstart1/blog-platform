## 一、需求合理度七维度评估

| 维度 | 评分 | 说明 |
|:------|:----:|:-----|
| **真实价值** | 4 | 个人博客是内容创作的基础需求，具有普适性。但对于"个人"博客系统而言，多用户体系、评论审核等需求略超个人场景，存在范围膨胀风险。 |
| **业务目标** | 3 | 目标较模糊——未明确是"学习项目"还是"上线产品"。缺少 DAU/文章量/用户量等量化指标。建议补充目标定位。 |
| **受众规模** | 3 | 作为"个人博客"，初始受众为博主本人+少量读者。多作者+访客注册功能指向了多人平台方向，受众定位不清晰。 |
| **技术可行** | 5 | Java Spring Boot + Vue 3 + MySQL + Redis 是成熟的通用技术栈，Markdown 编辑、全文搜索、评论系统等均有成熟方案，完全可行。 |
| **边界清晰** | 3 | 核心功能点已列出，但存在模糊地带：SEO 方案（SSG/SSR 二选一？）、多用户体系的权限粒度、评论审核规则、全文搜索引擎选型等未明确。 |
| **效果可验** | 4 | 各功能模块均可独立验证（CRUD、搜索、评论），验收标准可量化。 |
| **成本合理** | 4 | 单人全栈开发预估约 3-4 周，成本可控。若定位为学习项目则投入产出比更高；若定位为商业产品需重新评估。 |

| 统计项 | 数值 |
|:-------|:----:|
| **总分** | **26 / 35** |
| **均分** | **3.71** |

### 评估结论：**建议开发（有条件通过）**

需求整体可行，技术方案成熟。但需在开工前明确以下核心问题：
- 产品定位：学习项目 / 个人使用 / 多用户平台？
- SEO 方案选型：SSG（Nuxt.js）还是 SSR？
- 全文搜索引擎：MySQL LIKE / Elasticsearch / MeiliSearch？
- 评论审核策略：先发后审 / 先审后发？

## 二、用户角色与使用场景

### 2.1 用户角色矩阵

| 角色 | 权限摘要 | 核心诉求 |
|:-----|:--------|:---------|
| **博主 (Owner)** | 全部权限：文章管理、分类/标签管理、评论审核、用户管理、站点配置 | 高效创作、内容管理、读者互动 |
| **作者 (Author)** | 文章发布/编辑自己的文章、草稿管理 | 在博客平台发表文章 |
| **访客 (Visitor)** | 浏览文章、搜索、提交评论（需审核） | 阅读、发现内容、参与讨论 |
| **管理员 (Admin)** | 系统配置、用户管理、评论管理、数据统计 | 系统运维、内容审核、用户治理 |

### 2.2 核心使用场景

| 编号 | 场景 | 角色 | 描述 |
|:-----|:-----|:-----|:-----|
| S01 | 发布文章 | 博主/作者 | 使用 Markdown 编辑器编写文章，支持实时预览、插入图片、保存草稿、发布 |
| S02 | 管理内容 | 博主 | 对文章进行分类、打标签、编辑/删除，管理草稿箱 |
| S03 | 浏览阅读 | 访客 | 按时间线/分类/标签浏览文章列表，查看文章详情，代码高亮 |
| S04 | 全文搜索 | 访客 | 输入关键词搜索文章标题和内容，支持搜索结果高亮 |
| S05 | 提交评论 | 访客 | 在文章底部提交评论，支持回复嵌套，提交后进入审核队列 |
| S06 | 审核评论 | 博主/管理员 | 查看待审核评论列表，通过/拒绝/删除评论 |
| S07 | 访客注册 | 访客 | 注册成为平台用户，获得评论免审或作者申请权限 |
| S08 | 多作者协作 | 博主/作者 | 博主邀请/管理作者，作者在自己的权限范围内创作 |
| S09 | SEO 访问 | 搜索引擎爬虫 | 服务端渲染/静态生成页面，支持 TDK、sitemap、结构化数据 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 说明 |
|:-----|:------|:-----|
| 首页加载时间 (FCP) | < 1.5s | 含文章列表首屏渲染 |
| 文章详情页加载 | < 1s | 含 Markdown 渲染 + 代码高亮 |
| 搜索响应时间 | < 500ms | 基于索引搜索，非全表扫描 |
| 接口响应时间 (P95) | < 200ms | 常规 CRUD 接口 |
| 并发支持 | ≥ 100 QPS | 初期足够，可水平扩展 |

### 4.2 安全要求

| 要求 | 说明 |
|:-----|:-----|
| XSS 防护 | Markdown 渲染需过滤危险标签，评论内容 HTML 转义 |
| CSRF 防护 | 前端携带 Token，后端校验 |
| SQL 注入防护 | MyBatis-Plus 参数化查询 |
| 密码加密 | BCrypt 加密存储 |
| 接口鉴权 | JWT + 角色权限中间件 |
| 评论防刷 | 频率限制（Redis 实现），验证码 |
| 文件上传安全 | 类型校验、大小限制、重命名存储 |

### 4.3 SEO 要求

| 要求 | 说明 |
|:-----|:-----|
| TDK 动态设置 | 每个页面独立 title / description / keywords |
| sitemap.xml | 自动生成，提交搜索引擎 |
| 结构化数据 | Article / BreadcrumbList Schema.org 标记 |
| SSR/SSG | 首版建议 SSG（Nuxt.js 或 VitePress），文章更新时触发重新生成 |
| URL 规范 | 语义化 URL，如 `/article/{slug}` |

### 4.4 可维护性

| 要求 | 说明 |
|:-----|:-----|
| 前后端分离 | RESTful API，独立部署 |
| 接口文档 | Swagger / Knife4j 自动生成 |
| 日志规范 | 统一日志格式，关键操作记录 |
| 配置管理 | 环境变量分离（dev/test/prod） |
