## 二、Bug 清单

### BUG-001：搜索页面递归栈溢出
- **严重级别**：P1（功能不可用）
- **页面**：http://localhost:3000/search
- **现象**：`RangeError: Maximum call stack size exceeded`
- **文件**：`blog-frontend/src/pages/Search.vue`
- **分析**：组件存在递归调用导致栈溢出，可能是响应式依赖循环或组件递归渲染

### BUG-002：编辑文章时 ByteMD 报错
- **严重级别**：P1（功能不可用）
- **页面**：编辑 content 为 null 的文章时
- **现象**：`TypeError: Cannot read properties of null (reading 'split')` — ByteMD 的 `setValue` → `splitLines` 对 null 调用 split
- **文件**：`blog-admin/src/pages/ArticleEditor.vue`
- **分析**：文章 content 存储为 null（如文章 #1 "测试"），ByteMD 编辑器初始化时未对 null 做防御

### BUG-003：文章详情页已审核评论不显示
- **严重级别**：P1（功能不完整）
- **页面**：http://localhost:3000/article/:slug
- **现象**：评论已审核通过（API 返回 APPROVED），但详情页始终显示"评论 (0)"
- **根因**：`ArticleDetail.vue` 的 `onMounted()` 中 `fetchComments()` 未等待 `fetchArticle()` 完成，此时 `article.value` 为 null，`fetchComments()` 中 `if (!article.value) return` 直接返回，评论从未被加载
- **文件**：`blog-frontend/src/pages/ArticleDetail.vue` 第 202-205 行

```typescript
onMounted(() => {
  fetchArticle()    // async 未 await
  fetchComments()   // 此时 article.value 为 null，直接 return
})
```

- **修复建议**：在 `fetchArticle()` 成功回调中调用 `fetchComments()`

## 四、测试结论

- **核心流程可用**：文章发布 → 读者浏览 → 评论提交 → 后台审核 全链路打通
- **3 个 P1 缺陷待修**：搜索页崩溃、编辑 null content 文章崩溃、评论显示缺失
- **建议**：修复 3 个 Bug 后进行第二轮回归测试
