## 三、上线步骤

### 阶段 1：环境准备（预计 10 分钟）

```bash
# 1.1 创建部署目录结构
sudo mkdir -p /opt/blog/blog-server
sudo mkdir -p /opt/blog/blog-frontend/dist
sudo mkdir -p /opt/blog/blog-admin/dist
sudo mkdir -p /opt/blog/blog-nuxt/.output/public
sudo mkdir -p /data/blog/img
sudo mkdir -p /var/log/blog

# 1.2 设置目录权限
sudo chown -R app:app /opt/blog
sudo chown -R app:app /data/blog
sudo chmod 755 /data/blog/img

# 1.3 创建应用用户（如不存在）
sudo useradd -m -s /bin/bash app 2>/dev/null || true
```

### 阶段 2：数据库变更（预计 5 分钟）

```bash
# 2.1 备份现有数据库（如有）
mysqldump -u root -p --databases blog > /tmp/blog_backup_$(date +%Y%m%d_%H%M%S).sql

# 2.2 执行 DDL 脚本
mysql -u root -p < docs/blog/ddl/blog-ddl.sql

# 2.3 验证表结构
mysql -u root -p -e "USE blog; SHOW TABLES;"
# 预期输出 8 张表: user, category, tag, article, article_tag, comment, article_version, link

# 2.4 验证 FULLTEXT 索引（确认 ngram parser）
mysql -u root -p -e "SHOW CREATE TABLE blog.article\G" | grep -i fulltext
# 预期: FULLTEXT KEY `ft_title_content` (`title`,`content`) WITH PARSER ngram

# 2.5 验证初始博主数据
mysql -u root -p -e "SELECT id, username, role FROM blog.user WHERE username='admin';"
# 预期: 存在 admin/OWNER 记录
```

### 阶段 3：后端部署（预计 5 分钟）

```bash
# 3.1 构建后端 JAR
cd blog-server
mvn clean package -DskipTests -Pprod
# 产物: target/blog-server.jar

# 3.2 上传 JAR 到服务器
scp target/blog-server.jar app@<server-ip>:/opt/blog/blog-server/

# 3.3 上传配置文件
scp src/main/resources/application-prod.yml app@<server-ip>:/opt/blog/blog-server/

# 3.4 启动后端服务
ssh app@<server-ip> << 'EOF'
cd /opt/blog/blog-server

# 停止旧服务（如存在）
pkill -f blog-server.jar 2>/dev/null || true
sleep 2

# 启动新服务
nohup java -jar \
  -Xms512m -Xmx1024m \
  -Dspring.profiles.active=prod \
  -Dserver.port=8080 \
  blog-server.jar > /var/log/blog/server.log 2>&1 &

# 等待启动
sleep 10

# 健康检查
curl -s http://127.0.0.1:8080/api/health || echo "❌ 后端启动失败，请检查日志"
EOF
```

### 阶段 4：前端构建与部署（预计 8 分钟）

```bash

# 4.2 管理后台构建

# 产物: .output/public/

# 4.4 上传静态文件到服务器
scp -r blog-frontend/dist/* app@<server-ip>:/opt/blog/blog-frontend/dist/
scp -r blog-admin/dist/* app@<server-ip>:/opt/blog/blog-admin/dist/
scp -r blog-nuxt/.output/public/* app@<server-ip>:/opt/blog/blog-nuxt/.output/public/
```

## 四、灰度策略

由于是个人博客系统、IP 直连、单机部署，本次采用**一次性全量上线**而非灰度发布。

| 策略项 | 内容 |
|:-------|:-----|
| 上线方式 | 全量部署（单机） |
| 切流方式 | Nginx reload 原子操作 |
| 用户感知 | 首次部署无历史用户，无感知 |
| 后续迭代 | 可考虑蓝绿部署（切换 JAR 包 + Nginx upstream） |

### 后续迭代灰度建议

```nginx
# 蓝绿部署 upstream 配置（后续迭代使用）
upstream blog_backend {
    server 127.0.0.1:8080 weight=90;  # 蓝（当前版本）
    server 127.0.0.1:8081 weight=10;  # 绿（新版本）
}

# 切换时修改 weight 实现灰度切流
```

## 五、回滚方案

### 5.1 回滚触发条件

| 条件 | 级别 | 处理 |
|:-----|:----:|:-----|
| 首页 502/504 | 🔴 严重 | 立即回滚 |
| 文章发布 500 | 🔴 严重 | 立即回滚 |
| 数据库连接失败 | 🔴 严重 | 立即回滚 |
| 管理后台白屏 | 🟡 中等 | 30 分钟内修复或回滚 |
| 搜索超时 | 🟡 中等 | 降级为简单 LIKE 查询 |
| SSG 页面 404 | 🟢 低 | 记录，后续修复 |

### 5.2 回滚步骤（代码回滚）

```bash
# 步骤 1：回滚后端 JAR
ssh app@<server-ip> << 'EOF'
# 停止当前服务
pkill -f blog-server.jar

# 恢复上一版本 JAR（假设备份在 /opt/blog/backup/）
cp /opt/blog/backup/blog-server-$(date -d 'yesterday' +%Y%m%d).jar /opt/blog/blog-server/blog-server.jar

# 启动旧版本
nohup java -jar \
  -Xms512m -Xmx1024m \
  -Dspring.profiles.active=prod \
  /opt/blog/blog-server/blog-server.jar > /var/log/blog/server.log 2>&1 &
sleep 10
curl -s http://127.0.0.1:8080/api/health
EOF

# 步骤 2：回滚前端静态文件
ssh app@<server-ip> 'cp -r /opt/blog/backup/frontend-dist-$(date -d "yesterday" +%Y%m%d)/* /opt/blog/blog-frontend/dist/'
ssh app@<server-ip> 'cp -r /opt/blog/backup/admin-dist-$(date -d "yesterday" +%Y%m%d)/* /opt/blog/blog-admin/dist/'

# 步骤 3：重载 Nginx
ssh app@<server-ip> 'sudo nginx -s reload'

# 步骤 4：验证
curl -s -o /dev/null -w "%{http_code}" http://<server-ip>/
curl -s -o /dev/null -w "%{http_code}" http://<server-ip>/admin/
```

### 5.3 数据库回滚（如需）

```sql
-- 场景 1：DDL 变更回滚（如有新增字段）
-- 在事务中执行反向 DDL
ALTER TABLE article DROP COLUMN new_field;

-- 场景 2：数据回滚（使用上线前备份）
-- 恢复备份
mysql -u root -p blog < /tmp/blog_backup_YYYYMMDD_HHMMSS.sql
```

### 5.4 功能开关降级

| 进程存活 | Java 进程 | 不存在 | `ps aux | grep java` |

### 6.2 简易监控脚本（crontab）

```bash
# 添加到 crontab，每 1 分钟检查一次
# crontab -e
* * * * * /opt/blog/scripts/health-check.sh

# health-check.sh 内容
#!/bin/bash
HEALTH_URL="http://127.0.0.1:8080/api/health"
HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" $HEALTH_URL)

if [ "$HTTP_CODE" != "200" ]; then
    echo "[$(date)] ❌ 后端健康检查失败，HTTP $HTTP_CODE" >> /var/log/blog/health.log
    # 可选：发送告警通知
fi

# 检查磁盘空间
DISK_USAGE=$(df /data/blog | tail -1 | awk '{print $5}' | sed 's/%//')
if [ "$DISK_USAGE" -gt 80 ]; then
    echo "[$(date)] ⚠️ 磁盘使用率 ${DISK_USAGE}%" >> /var/log/blog/health.log
fi
```

## 附录 A：Nginx 完整配置

> 保存为 `/etc/nginx/conf.d/blog-platform.conf`

```nginx
# ============================================
# 博客平台 Nginx 配置 — IP 直连
# ============================================

# 限流配置
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/s;
limit_req_zone $binary_remote_addr zone=comment_limit:10m rate=3r/m;

server {
    listen 80;
    server_name _;

    access_log /var/log/nginx/blog-access.log main;
    error_log  /var/log/nginx/blog-error.log  warn;

    client_max_body_size 10m;
    server_tokens off;

    # ----------------------------------------
    # 1. 管理后台 — /admin/ base path (SPA)
    # ----------------------------------------
    location /admin {
        alias /opt/blog/blog-admin/dist/;
        index index.html;
        try_files $uri $uri/ /admin/index.html;

        location ~* /admin/assets/.*\.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf)$ {
            alias /opt/blog/blog-admin/dist/;
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # ----------------------------------------
    # 2. 博客前端 — 根路径 (SPA)
    # ----------------------------------------
    location / {
        root /opt/blog/blog-frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;

        location ~* ^/assets/.*\.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf)$ {
            root /opt/blog/blog-frontend/dist;
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # ----------------------------------------
    # 3. Nuxt SSG 静态页面
    # ----------------------------------------
    location /ssg {
        alias /opt/blog/blog-nuxt/.output/public;
        index index.html;
        try_files $uri $uri/ /ssg/index.html;
    }

    # ----------------------------------------
    # 4. 图片服务 — 本地磁盘
    # ----------------------------------------
    location /img/ {
        alias /data/blog/img/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        add_header X-Content-Type-Options "nosniff";

        location ~* \.(jpg|jpeg|png|gif|webp|svg)$ {
            alias /data/blog/img/;
            expires 30d;
        }
    }

    # ----------------------------------------
    # 5. 后端 API 代理
    # ----------------------------------------
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_connect_timeout 10s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;

        limit_req zone=api_limit burst=50 nodelay;
    }

    # ----------------------------------------
    # 6. Swagger/Knife4j 文档
    # ----------------------------------------
    location /doc.html {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
    location /swagger-resources {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
    location /v3/api-docs {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
    location /webjars {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }

    # ----------------------------------------
    # 7. 安全加固
    # ----------------------------------------
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }

    location = /sitemap.xml {
        root /opt/blog/blog-frontend/dist;
    }
    location = /robots.txt {
        root /opt/blog/blog-frontend/dist;
    }
}
```
